Build a Django Portfolio Website

A comprehensive, step-by-step guide to creating a clean and professional portfolio using the Django framework.

1. Introduction

This guide will walk you through building a complete portfolio web application from scratch using Django. The goal is to create a dynamic site where you can showcase your projects. We will cover setting up the project, defining the data model, creating views and templates, and styling the frontend.

2. Prerequisites & Setup

Before you begin, ensure you have the following installed:

  • Python 3.x
  • pip (Python package installer)

Follow these steps to set up your project environment and initial Django structure.

Step 1: Create a Virtual Environment

It's a best practice to create a virtual environment to isolate project dependencies.

# Create a directory for your project and navigate into it
mkdir django_portfolio && cd django_portfolio

# Create a virtual environment
python -m venv .venv

# Activate the virtual environment
# On Windows:
.venv\Scripts\activate
# On macOS/Linux:
source .venv/bin/activate

Step 2: Install Django and Pillow

With the virtual environment active, install Django and Pillow (for image handling).

pip install Django Pillow

Step 3: Create the Django Project and App

Now, create the main project container and a dedicated app for your portfolio.

# Create the project (don't forget the '.' at the end)
django-admin startproject portfolio_project .

# Create the portfolio app
python manage.py startapp portfolio

Step 4: Configure Settings

Open portfolio_project/settings.py and add your new app to the INSTALLED_APPS list:

'portfolio',

You should also configure your template directories and static/media file settings, but that is beyond the scope of this file-by-file guide.

3. Project Code (File by File)

Here is the complete code for each file in the project. We'll go through the models, views, URLs, templates, and styling.

portfolio/models.py

This file defines the Project model, which represents the structure of your project data in the database. Each class attribute maps to a database field.

# portfolio/models.py

from django.db import models

class Project(models.Model):
    title = models.CharField(max_length=100, help_text="The title of the project.")
    description = models.TextField(help_text="A detailed description of the project.")
    # The 'upload_to' path is relative to your MEDIA_ROOT setting
    image = models.ImageField(upload_to='portfolio/images/', help_text="An image representing the project.")
    technologies = models.CharField(max_length=200, help_text="Comma-separated list of technologies used.")
    link = models.URLField(blank=True, help_text="A URL to the live project or its repository.")

    class Meta:
        verbose_name_plural = "Projects"
        ordering = ['title']

    def __str__(self):
        return self.title

portfolio/views.py

Views handle the logic for web requests. These functions fetch data from the database via the Project model and pass it to the templates for rendering.

# portfolio/views.py

from django.shortcuts import render, get_object_or_404
from .models import Project

def project_list(request):
    """
    Displays a list of all projects.
    """
    projects = Project.objects.all()
    context = {
        'projects': projects
    }
    # Assumes a project-level 'templates' directory is configured in settings.py
    return render(request, 'project_list.html', context)


def project_detail(request, pk):
    """
    Displays a single project identified by its primary key (pk).
    """
    project = get_object_or_404(Project, pk=pk)
    context = {
        'project': project
    }
    return render(request, 'project_detail.html', context)

portfolio/urls.py

This file contains URL patterns specific to the portfolio app. Creating this file makes your app's URLs modular and reusable. You will need to create this file manually inside your portfolio app directory.

# portfolio/urls.py

from django.urls import path
from . import views

# Namespacing URLs for the 'portfolio' app
app_name = 'portfolio'

urlpatterns = [
    # Root URL of the app maps to the project list view
    path('', views.project_list, name='project_list'),

    # URL for detail view, capturing the project's primary key
    path('<int:pk>/', views.project_detail, name='project_detail'),
]

portfolio_project/urls.py

This is the main URL configuration file for the entire project. It delegates requests to the app-specific urls.py file we created above.

# portfolio_project/urls.py

from django.contrib import admin
from django.urls import path, include

# Import settings and static for serving media files during development
from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    path('admin/', admin.site.urls),

    # Include URLs from the 'portfolio' app at the project's root
    path('', include('portfolio.urls')),
]

# Helper for serving media files (e.g., project images) in development
if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)

templates/project_list.html

This template inherits from base.html and displays a grid of all your projects. It loops through the `projects` data passed from the view.

{% raw %}{% extends "base.html" %}

{% block content %}
    <section class="project-gallery">
        <h1>My Projects</h1>
        <div class="project-grid">
            {% for project in projects %}
                <div class="project-card">
                    <a href="{% url 'portfolio:project_detail' project.id %}">
                        {% if project.image %}
                            <img src="{{ project.image.url }}" alt="Image for {{ project.title }}">
                        {% endif %}
                        <div class="card-body">
                            <h3>{{ project.title }}</h3>
                        </div>
                    </a>
                </div>
            {% empty %}
                <p>There are no projects to display at this time.</p>
            {% endfor %}
        </div>
    </section>
{% endblock content %}{% endraw %}

4. Running the Application

With the code in place, you need to create the database schema and run the development server.

Step 1: Create Database Migrations

This command analyzes the changes to your models and creates migration files.

python manage.py makemigrations

Step 2: Apply Migrations to the Database

This command applies the migrations, creating the necessary database tables.

python manage.py migrate

Step 3: Create a Superuser (Optional)

To add projects via the Django admin interface, create a superuser.

python manage.py createsuperuser

Step 4: Run the Development Server

Start the server to see your site in action.

python manage.py runserver

Your portfolio will be available at http://127.0.0.1:8000/. You can log into the admin panel at /admin/ to start adding your projects.

5. Conclusion

You have successfully built a functional portfolio website with Django. You have a solid foundation with a project list, detail pages, and a clean design.

From here, you can customize the styles, add more fields to your Project model, or even add new features like a contact form or a blog.
